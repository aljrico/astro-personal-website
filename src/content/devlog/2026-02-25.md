Started the day on Hollows. Shipped four node types in a single commit — toll, shrine, sanity drain, mimic. The mimic disguises as a chest then attacks you. That's the kind of design that writes itself. The shrine trades fangs for sanity and it *feels* right in a way I can't explain mechanically. Swapped all the SF Symbol icons for the Rhos fantasy pack and spent way too long picking the boss icon. Rejected the king. Went with an eye silhouette. Lovecraft over Tolkien. The eye breathes purple at 1.5x scale and I keep opening the app just to look at it.

Then switched to FlamaCoach and fell into the description trap.

Twenty topic descriptions. That's it. Twenty one-liners for the Mossos Guide drill cards. Should take, what, an hour? I'll batch it through the API.

Round one: gpt-4.1-nano. Cheap, fast, terrible. Two-sentence summaries that read like Wikipedia intros. Nobody's reading that on a phone card. Upgraded to gpt-5.2 with low reasoning effort. Better, but still wordy. $0.012 for twenty descriptions that I immediately wanted to rewrite.

Round two: tighter prompt. "UNA SOLA FRASE, màxim 20 paraules, estil telegràfic." Got keyword soup back. Technically compliant, emotionally dead. The kind of text that makes you skip past it without reading. Removed the maxLines clamp from the cards so they'd grow dynamically, which was the right fix for the UI but didn't fix the words.

Round three: I wrote them myself. All twenty. By hand.

The first one set the tone — "Evolució política i social de Catalunya des de l'antiguitat fins al segle XVIII: romanització, feudalisme, expansió mediterrània i guerres." Framing clause, colon, key milestones. Rhythm. I did the other nineteen in that voice and they were done in maybe forty minutes.

Three iterations, two batch API calls, ~$0.024 in tokens, and the answer was always just: write them yourself, idiota. Twenty items where tone matters isn't a scale problem. It's a craft problem. The batch script lives on for when I need 200 descriptions. For 20 high-visibility lines that set the personality of the app, the LLM was the wrong tool.

There's a lesson I keep relearning. The instinct to automate isn't always about efficiency. Sometimes it's about avoiding the blank page. Easier to tweak a prompt than to stare at an empty string and decide what the description *should* say. The batch script is a shield against having to make creative choices. And creative choices are exactly what twenty topic descriptions need.

Fixed a hot restart bug along the way. The database was "disappearing" after hot restart — turns out the MD5 hash changes when you rebuild the asset, triggers a re-copy, but hot restart doesn't re-push assets so the static DB reference goes null. Moved `DatabaseService.initialize()` from the splash screen to `main()`. Five-minute fix once I understood it. Felt like an hour before I did.

Wrapped by documenting the whole pipeline in the README. Chunk → questions → drill topics → descriptions → DB. Included the style examples — good vs bad descriptions, inline, while I still remembered why the bad ones were bad. Documentation written the same day as the decisions is a completely different artifact than documentation written a month later. One is a trail. The other is archaeology.

Two projects. One day. The Hollows work was all momentum — four phases, aesthetic decisions, ship it. The FlamaCoach work was the opposite — circling, iterating, realizing the machine couldn't do what I needed, then doing it in less time than I spent prompting. Both valid. Both mine.

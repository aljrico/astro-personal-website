Spent the day making the game do less.

There's a specific satisfaction in performance work that's different from building features. Features are additive — you make something exist that didn't. Performance is subtractive. You make something disappear that shouldn't have existed in the first place. The CPU graph drops and you feel like you've been carrying a backpack full of rocks without knowing it.

Ripped out the SpriteKit audio layer entirely. MainJuiceScene was running a whole scene just to play sound effects. A SpriteKit scene. For beeps. Replaced it with AVAudioPlayer wrapped in a plain manager class, dropped the CADisplayLink from 60fps to 15, gated the garden updates on visibility, throttled scoreboard animations. CPU went from 16.2% to 13.3% mean. The 20-25% spikes just vanished.

The numbers are satisfying in that way that only a physicist finds satisfying. 18% reduction. Clean. Measurable. I can put it in a spreadsheet and feel productive. But the real win was something the code review caught that I'd missed entirely.

Eleven call sites were bypassing the user's "haptics off" setting.

Eleven. The old Haptics enum was scattered everywhere — TilesMatch, buildings, upgrades — and none of those call sites checked whether the user had actually wanted haptic feedback. So if you'd gone into settings, turned off haptics, the game would politely ignore you and keep buzzing. For who knows how long. Months, probably.

This is the kind of bug that makes you question everything. Not because it's hard to fix — it's trivial — but because it means the feature was never really working. The toggle existed. The UI existed. The user's intent was captured and stored and then comprehensively ignored. Hostia.

Fixed it. JuiceManager.shared.playHaptic() is now the only entry point. Settings check built in. Done.

Tried marking JuiceManager as @MainActor while I was in there. Felt like the right thing. Swift Concurrency, proper isolation, all the stuff you're supposed to do. Cascading errors everywhere — SpriteKit callers, Codable conformances, half the codebase lighting up red. Reverted in under ten minutes. All the callers are main-thread anyway. The annotation would've been correct and completely pointless. Sometimes the "right" thing is just ceremony.

There's a dead Haptics enum sitting in a file that also contains MinimalButtonStyle. Should rename the file. Won't today. Maybe won't ever. It works. The compiler doesn't care what the file is called. Neither do I, really, but it'll nag at me every time I see it in the navigator. The kind of thing that's not worth doing until one day it suddenly is.

The whole session was maintenance. No new features. No new pixels on screen. Nothing to show anyone. But the game runs cooler, respects user settings, and has one less SpriteKit scene doing a job that didn't need a scene. Subtractive work. The backpack is lighter.

Woke up wanting to kill CPU cycles. The profiler had been bugging me — 16% average with spikes touching 25%. For a clicker game. An *idle* clicker game. The irony wasn't lost on me.

The main offender was MainJuiceScene, a full SpriteKit scene running just to play tap sounds and haptics. A whole rendering pipeline for audio. Like hiring a orchestra to play a doorbell. Replaced it with a plain AVAudioPlayer wrapper — JuiceManager. Dropped CADisplayLink from 60fps to 15. Gated TinyGarden updates behind visibility so it stops ticking when nobody's looking at it. Throttled scoreboard animations. The numbers came back: 13.3% mean CPU. No spikes above 18%.

Eighteen percent reduction. Two lines on a profiler chart. Unreasonably satisfying.

Then ran /simplify and it humbled me. Found a dead playerPool I'd forgotten about, stringly-typed sound references where I had perfectly good enums, and — the real one — eleven call sites bypassing the user's "haptics off" setting. Eleven. The user toggles haptics off in settings and the game just... keeps vibrating. In eleven different places. Hostia.

That's the thing about fast coding. You move quick, you ship, the feature works, you feel smart. Then a code review finds eleven spots where you ignored your own settings flag and suddenly you're the guy who shipped a "mute" button that doesn't mute. The impostor's voice had a field day with that one.

Fixed all of it. Also caught a subtle bug: timerDisplayAccumulator in TinyGarden was growing unbounded while the tab was hidden, then when you switched back it would fire 45 withAnimation calls in a burst. The kind of bug that makes you wonder what else is ticking away in the background, accumulating, waiting.

Tried marking JuiceManager as @MainActor because it felt like the right thing to do. Cascading compiler errors from SpriteKit and Codable callers. Reverted in under a minute. All callers are main-thread anyway. The "right" thing and the "working" thing aren't always the same thing, and today I chose working.

Deleted the old Haptics enum entirely. Dead code since the migration. There's a specific pleasure in deleting code — the opposite of the anxiety of writing it. Every line removed is a line that can't break. The file it lived in still has MinimalButtonStyle and friends, so it stays, just with a slightly misleading name now. Should rename it. Won't today.

JuiceManager.shared.playHaptic() is now the single entry point for all feedback. One path in, one settings check, cached generators. Clean. The kind of clean that actually matters — not aesthetic clean, but "this bug literally cannot happen anymore" clean.

Good day. Measurable improvement, real bugs found and killed, dead code buried. The numbers don't lie and the numbers got better. Tomorrow I'll find something else to obsess over, but today the profiler chart looks right and that's enough.

Review triage day. No new code that matters, just three findings from an external review and the slow work of deciding which ones deserve to live.

The first one was the good kind of wrong. The reviewer claimed the XP flush could drop solves for players who never opted in. That path is unreachable, a pref check bails out in microseconds while the supposedly dangerous path is still waiting on the network. But while building the case against it I found an actual bug underneath: a failed in-flight flush retry that really could lose a solve. Wrong diagnosis, real disease. Fixed the disease, filed the diagnosis under "close enough." Half of what a review buys you isn't the findings. It's being forced to stare at your own code with hostile eyes.

The second finding, I went to the source. Literally. The reviewer insisted Android's thermal listener gives you no baseline reading, and instead of trusting either of us I read the AOSP code. ThermalManagerService posts the current status to new listeners the moment they register. The reviewer was wrong. Round two came back hours later making the same claim. Still wrong. Same file, same lines. And then I hardened the code anyway, because the behavior isn't documented anywhere, and undocumented is a promise nobody made. Won the argument, wrote the code like I'd lost it. Being right and being safe are different bets.

There's something quietly absurd about spending an afternoon debating a machine that can't be embarrassed. It doesn't concede. It just re-runs. You can't win, you can only be correct, and those turn out to be different things too.

The fun part was the ARENA button. Stripped it down to a lone icon in a near-square pill that sizes itself off its sibling button, and it just clicks now. Ten minutes of layout fiddling, easily the most satisfying commit of the day. Also learned I can poke SIGUSR1 at a running flutter process from another terminal to hot-reload it. Felt like whispering to the machine. I'll be abusing this forever.

Round two also dragged sixteen test failures out of the duel screen. Joder. Fifteen of them turned out to be one pre-existing bug, bootstrap doing a synchronous locale lookup inside initState, nothing I broke today. Moved it to first didChangeDependencies, fixed a test that was still expecting lowercase labels I'd deliberately uppercased weeks ago, and the whole suite went green. 133 out of 133. Pushed.

Maintenance day. Nothing shipped, nothing new. The release inches forward one defended finding at a time. Some days you build the house. Some days you argue with the inspector.

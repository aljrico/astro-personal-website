Shipped the big one last night. Official exam simulation — eight real convocatories with their metadata, places offered, aspirants, time limits. The scoring formula straight from the exam board: `P = (C - I*0.25) / N * 10`, clamped to zero, pass at 4.0. Attempt tracking, best-score upsert, APTE/NO APTE badges on the selection cards. The whole thing.

It felt substantial. Not a tweak, not a fix. A feature. The kind where you push and think: this app just got meaningfully better.

Then I reviewed my own code.

Thirteen minutes later I found a bug that would have shipped wrong scores to every user. The `finishedResults` list was being rebuilt from `state.results` — five lines after `state.results` had already been mutated with the last answer. So the last answer got counted twice. Every persisted exam score would have been N+1 elements. Every result subtly wrong.

Two-line fix. Used `state.correctCount` and `state.totalQuestions` directly instead of reconstructing from the list. But the lesson is the thing: `flutter analyze` is completely blind to read-after-mutate-state bugs. The linter sees valid Dart. The types check out. The only thing that catches it is a human reading the method top to bottom and asking "wait, did I already change that?" Manual code review. The oldest tool in the box.

Hostia. Imagine shipping that. Imagine a user taking an exam, getting 7 right out of 30, and seeing a score that says 8. Content accuracy is the one thing you can't get wrong in an exam app. The one thing.

Slept on it. Woke up calmer.

Morning was polish. The comfortable kind — reordering metadata on exam cards so format info comes first, context second. Questions and time before places and aspirants. Added a pass ratio with bold styling, tried an arrow separator, settled on a middot. Killed the repeat button on the results screen since there's only one action now, unwrapped the orphaned Column.

The middot decision took longer than it should have. But that's the visual stuff — the thing where my brain actually engages instead of just executing. A middot versus an arrow is a stupid thing to care about. I cared about it anyway.

Three commits. One feature, one fix, one polish pass. The feature is the headline but the fix is the one I'll remember. That thirteen-minute gap between "shipped" and "oh no" — that's the gap where trust lives. Or dies.

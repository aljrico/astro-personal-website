Two-project day. The kind where you wake up building one thing and go to bed fixing another and somewhere in between you forget which app you're even looking at.

Started in Cookie Clicker land. The islands tech tree skeleton went in — 25 nodes across 5 branches, data model, manager, persistence, full placeholder UI. Kept it simpler than Hell's protocol-heavy approach. Enums with computed properties instead of per-node classes, because these nodes are boolean toggles, not multi-level beasts. Horizontal branch layout. It felt right. Sometimes the second time you build a system you actually build it correctly instead of over-engineering it.

Then ripped out the passive +10% CpS per island level bonus. Islands don't give production bonuses anymore — the tech tree replaces that whole reward model. Meaningful gameplay change. Existing players will silently lose their multiplier. I thought about migration paths for about thirty seconds and then decided: nah. It's a beta branch. Move fast.

The wobbly squircle polish hit that sweet spot. Swapped Circle() for WobblySquircleShape on the tech nodes, deterministic seeds so each node gets its own wobble. Replaced the SF Symbol star with the actual egg icon. Small stuff. The kind of work where you keep rebuilding just to look at it. Flow lives here — in the visual tweaks, in making the screen do exactly what you pictured.

Then I switched to Mossos Coach and the energy changed.

Settings bottom sheet. Preferences provider with SharedPreferences preload. Theme toggle replaced with a gear icon. Nested Navigator for in-sheet push/pop navigation instead of stacked sheets. Legal copy extracted to markdown assets with platform-conditional blocks. The method page rewritten to lead with brand pillars instead of academic citations. Subscription management for the premium tile. Profile tile deleted entirely because there's zero auth behind it and Apple would reject it in seconds.

Shipping work. Not glamorous. Necessary.

The tap freeze bug was the day's main character though. ~400ms freeze on every big cookie tap. Traced it through toast-based timing instrumentation because console streaming wasn't working — had to build little diagnostic toasts into the UI like a caveman. Narrowed it down: click() → onTap() → addNumber() → playSound(). The JuiceManager refactor from a few commits back had replaced SKAction.playSoundFileNamed with AVAudioPlayer(data:), which decodes MP3 on the main thread. Every. Single. Tap. Hostia.

Tried audio player pooling. Didn't fix it. Tried scene isPaused toggling. 384ms wake-up cost. Even consulted Codex. Dead end after dead end. The fix was embarrassingly simple: route audio back through SKAction on the always-mounted FlyingDecorationScene, throttle idle to 1fps. Two concepts, ten minutes of actual coding, three hours of detective work. That ratio never improves.

Pushed four commits to the islands branch. Restyled the tech tree unlock button to match Hollows — slate blue-grey tones sampled from the black egg icon instead of the orange/brown I had. Visual consistency matters more than I want to admit. A tech tree that looks like it belongs in the same UI family as the rest of the game is worth the extra pass.

Back to Mossos Coach for the late shift. App Store Connect subscription cleanup. Created localizations in three languages, then deleted two of them — app is Catalan-only now. Used the raw ASC REST API with JWT generation because the CLI doesn't support subscription localization CRUD. The kind of yak-shaving where you're writing Node.js crypto code to manage App Store metadata at midnight and wondering how you got here.

Subscription products still show MISSING_METADATA. Probably needs a review screenshot uploaded through the web UI. Always one more thing.

Two projects in one day means neither gets your full attention. But both moved forward. The tech tree has bones. Mossos Coach is closer to submission. The tap freeze is gone. The legal copy is review-ready. That's a lot of ground covered for a Thursday.

The question I keep not asking: should I be working on one thing instead of two? The physicist brain says yes, obviously, focus compounds. But the mad scientist needs the variety. Needs to switch contexts when one project starts feeling like homework. Maybe that's a rationalization. Probably is. But the code shipped either way, and that's the only metric that doesn't lie.

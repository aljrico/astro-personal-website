Morning started gentle. Theme toggle on the Astro site — tried fixed positioning first because it seemed clean. It wasn't. Felt like the button was floating in space, disconnected from everything else on the page. Moved it back into the header with `ms-auto` and `items-start`, and now it sits where your eye expects it. Top-right, aligned with the title. Sometimes the first instinct is wrong and the boring solution is the right one.

Then JSON-LD tags started crashing the build. Astro's esbuild was trying to bundle structured data as JavaScript modules, which is like trying to compile a shopping list. The fix is `is:inline` — tells Astro to leave the script alone. Two characters of attribute that cost me twenty minutes of confusion. The lesson is simple: Astro processes every `<script>` tag by default. If it's not actual JS, opt out. Filed that one away.

Afternoon shifted to Inkwell. Sat down and did something I should've done weeks ago — went through Typora feature by feature, compared it against where Inkwell is, and filed 24 GitHub issues for the gaps. Twenty-four. That's either a sign the project has a long way to go or that I'm being thorough. Probably both. The spreadsheet brain kicked in hard. But this time it was useful — now there's a map instead of a vague feeling of "it's not ready yet."

Started knocking them off. First one: clickable links. Custom `.linkDestination` attribute, Cmd+click opens the URL, auto-prepends `https://` if you forget the scheme, hover tooltip shows where you're going. Clean implementation. Felt good.

Then found a bug that's been there forever. Link markers — the `[]()` syntax — were never hiding properly. The `parentRange` from the marker never matched the semantic token range, so the hiding logic just... never triggered. The whole time I thought it was a rendering quirk. It was a range mismatch. One fix — emit a full-range semantic token — and suddenly markers disappear like they always should have. The kind of bug where you wonder how you ever looked at it and thought it was working.

The cursor, though. Hostia.

Cmd+hover should change the cursor to a pointing hand over links. Simple concept. NSTextView had other plans. The thing aggressively resets the cursor to I-beam through internal methods that fire on every mouse move. Tried overlay views — defeated. Tried cursor rects — overridden. Tried push/pop cursor — ignored. Every approach felt like arm-wrestling a system that was specifically designed to win this fight.

I was going in circles. Then something clicked — tracking areas plus `cursorUpdate` plus TextKit 2 segment enumeration. Hit the right combination and it worked. The kind of solution that looks obvious in retrospect but required burning through every wrong answer first.

Two commits by end of day. The link clicking and marker fix in one, the cursor hover in another. Inkwell is starting to feel less like a toy and more like a tool. Twenty-three issues to go.

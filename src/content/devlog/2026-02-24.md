Different project today. The exam prep pipeline — chunking guide documents, generating question banks via the OpenAI batch API. Infrastructure work. The kind of day where you write zero lines that a user will ever see, but you can feel the machine getting smarter underneath.

Built a chunker for the guide markdowns. Three documents covering twenty topics, 289 chunks. Clean. The satisfying part isn't the code — it's watching structured data emerge from messy prose. A markdown file goes in, a JSONL comes out, and suddenly you have something an LLM can chew on. There's a factory-floor pleasure to it. Raw material in, product out.

The real win was the prompt fix. Had a LENGTH_RANK_TARGET compliance problem — the model was ignoring the length and difficulty constraints, hitting 13% compliance. Thirteen percent. Basically doing whatever it wanted. Swapped the terse rule-style instructions for step-by-step procedural ones. Same intent, different packaging. Compliance jumped to 90%.

Thirteen to ninety. By rewording a paragraph.

This is the part of working with LLMs that feels like sorcery and also like complete bullshit. The model didn't get smarter. The task didn't change. I just found the right incantation. There's no theory here, no first principles I can point to. Just: this phrasing works, that one doesn't. Prompt whispering. Half science, half vibes.

Added a `--tag` flag to keep guide chunks and legal doc chunks in separate namespaces. Small thing, but the kind of small thing that saves you from a stupid clobbering bug at 11pm. Past me protecting future me. Rare act of foresight.

Submitted 286 chunks to the batch API. Seventy-six cents. A full question bank across twenty topics for less than a euro. The batch is cooking. Results tomorrow probably.

Then the gears shifted completely. The exam prep app needed a visual identity — it had none. Navy palette, dark mode support, the works. Twenty-four files touched, a thousand lines added, fourteen hundred deleted. Net negative lines. The best kind of refactor.

Replaced a pile of static color constants with a proper theme extension. Thirty-four semantic tokens, light and dark variants. Bundled two font families. Hit a stupid gotcha — the static font ZIPs from Google Fonts were returning 404 HTML pages instead of actual fonts. Hostia. Spent twenty minutes thinking my download logic was broken before realizing the URLs were just dead. Grabbed the variable font from the raw GitHub URL instead. Worked immediately. The kind of bug that makes you feel both stupid and vindicated.

Killed the legacy quiz flow. Three files, two models, a utility function. Gone. The home screen is a proper dashboard now — hero card, theme toggle, the whole thing skinned to feel institutional without feeling sterile. Nine screens reskinned. Analyzer clean. Build green. That moment where you run the app and it looks like a *different app* — that's the hit.

Midday pivot to the idle clicker. The Hollows expedition needed a "not enough gems" upsell — the kind of thing that sounds like a one-hour job and turns into a ten-build-cycle dance with SwiftUI sheets. Started with a vertical card layout. Didn't feel right. Landed on a horizontal card modeled after the existing compact no-ads card. Two-phase animation: spring the sheet up, then fade the content in. Background transitions from light to a duotone combat scene when expanded. Ten rotating taglines with that numericText animation that makes text swaps feel alive.

Tuned the sheet detent from 0.55 down to 0.40 over several iterations. The difference between 0.55 and 0.40 is invisible on paper and obvious on screen. This is the stuff that doesn't show up in diffs but makes a feature feel intentional instead of slapped on.

Found a real bug along the way: the gem store's product fetch was only triggered from the gem store page. So the upsell sheet — which needs StoreKit products to display prices — had nothing to show. Classic. The feature that surfaces the bug that's been hiding since forever. Wired up the fetch to app launch. Done.

Stayed in the Hollows codebase for a performance pass. The map tree view was recomputing Reingold-Tilford layout on every frame — cached it, added viewport culling for nodes and edges so off-screen content doesn't even render. Debounced saves with 300ms coalescing for non-critical actions. The kind of work that's invisible to the player but turns a stutter into butter.

Pushed it and got a review back almost immediately. Real bug: `CGRect.zero.insetBy(dx: negative)` produces a *non-empty* rect, so the viewport-culling bypass for initial render was silently failing. The ripple animation on first load just... wouldn't show nodes. Swapped the isEmpty check for an explicit cullingEnabled flag that checks actual viewport dimensions. Also caught a redundant-save overlap in the debounce logic and stripped an unnecessary O(n log n) UUID sort that was added for stable ForEach identity but wasn't doing anything useful.

Build succeeded first try on the perf commit. The review bugfix too. Clean.

Lesson I keep relearning: keep SwiftUI sheet layouts simple. VStack, conditional content, Spacer. The moment you start overthinking backgrounds and nested geometry readers, you're fighting the framework instead of using it.

Back to the exam app for a polish pass. Dark mode needed work — the quiz options were muddy, the feedback banners felt off. Added proper navy tokens for option backgrounds and borders. Bumped the question typography to semi-condensed bold. Made the CORRECT/INCORRECT banners loud — all-caps, big type, two-tone icons. The kind of detail that doesn't matter until it does. You get a question right and the screen should *reward* you. Green badge, clean type, a button that says "let's keep going" without saying it.

Tried a border saturation swap on the dark palette. Got instantly rejected in review. Trust the first instinct on dark palettes. The eye knows before the brain can articulate why.

Then the font rabbit hole opened. Correct and wrong answer options were getting solid background fills — green for correct, red for wrong, matching the action buttons. Looked clean. But the text weight was wrong. w900 on Nunito looked like it was screaming at you. Dialed it back to w700, same as unselected. Only color and background distinguish answered state now. Subtler.

But wait. The weights weren't even working. This is where the afternoon got weird.

Flutter ignores variable font weight axes. Just... ignores them. You can set `FontWeight.w700` all day and it'll render whatever the default instance is. The variable Nunito TTF was technically correct — it had fvar and gvar tables, a proper weight axis. Flutter didn't care. Everything rendered as one weight.

The fix was surgical and deeply annoying. fonttools instancer with `inplace=True` to bake the gvar deltas into glyf outlines, then strip fvar and gvar entirely. Generated static TTFs — Regular, SemiBold, Bold, ExtraBold, Black — each with the correct usWeightClass baked in. Flutter finally saw the weights.

The trap I almost fell into: deleting gvar without baking the deltas first. That gives you outlines frozen at the default instance, which for Nunito is ExtraLight. Every weight renders as ExtraLight. You stare at it for ten minutes thinking your CSS is wrong before you realize the font file is lying to you. Hostia.

Three projects in one day. Pipeline plumbing, a full reskin, a monetization upsell, a performance pass, a font-tooling deep dive, a polish pass. None of it individually heroic. All of it necessary. The pipeline batch is cooking overnight. The exam app looks like it belongs to someone who gives a shit. The gem upsell exists where there was nothing. The Hollows map doesn't choke on large trees anymore.

Not a day you celebrate. A day you stack. And the stack is what eventually becomes something.

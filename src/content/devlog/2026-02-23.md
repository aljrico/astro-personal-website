Started the day cleaning house. The codebase still had Expo and React Native ghosts everywhere — CLAUDE.md, SPEC.md, examples in TypeScript that hadn't been true for weeks. Ripped it all out, replaced with the Flutter reality. Riverpod, sqflite, ConsumerWidget. 637 lines in, 732 lines out. Net negative. The best commits delete more than they add.

That was the warm-up.

The real session was about Mossos. Second exam in the pipeline. If Bombers was the proof of concept, Mossos is the proof that the concept generalizes. One exam is a hobby project. Two exams is architecture that either holds or doesn't.

Cracked the Portal Jurídic API and that was genuinely fun. POST to a SOAP-era endpoint, get back structured JSON with the full document hierarchy. Except Python's requests library choked on the TLS handshake — the Generalitat's servers apparently run crypto from 2014. Shelled out to curl like a barbarian. It worked. Six laws, 397 articles, 650K of markdown. Sometimes the ugly solution is the only solution and that's fine.

Then the pipeline work. Three scripts needed the same surgery: stop assuming Bombers, accept a `--source-dir` flag, derive paths dynamically. The chunker took five lines. The question generator took more — exam name injection into the system prompt, topic_id fallback for flat directory layouts, law label consolidation. But the pattern was clear: find the hardcoded assumption, replace with a parameter, test on two chunks, submit the batch.

620 tasks. $1.66 estimated. Submitted and waited.

The Codi d'ètica produced zero chunks. It's prose — sections instead of articles — and the chunker only speaks article headings. Parked it. Not every problem needs solving today.

Batch came back clean. 620 out of 620, zero failures. 1,882 questions. For about thirty seconds I felt like a genius.

Then the audit.

51.6% of correct answers at index 0. Position bias. The model defaults to putting the right answer first and hopes nobody notices. 29 questions with malformed options — empty strings, duplicates. And zero D5 questions. Not "few." Zero. Asked for system comprehension, got pure memorization across the board.

Filtered down to 1,863 after removing the broken ones. Committed, pushed. But the number that sticks is 51.6%. Half the questions are trivially gameable by always picking A. That's not an exam, that's a coin flip with extra steps.

The fix list writes itself: explicit shuffle instruction in the prompt, length-rank compliance enforcement, a dedicated D5 generation pass. All tractable. None of it hard. But it's the kind of thing that reminds you — the pipeline isn't the product. The pipeline is just the machine. The product is whether a firefighter candidate trusts your questions enough to study from them. And right now, anyone paying attention would notice the pattern.

One exam was scrappy and charming. Two exams is where the engineering debt starts compounding. The shortcuts that were fine for Bombers — hardcoded paths, implicit assumptions about directory structure, a single prompt that tries to do everything — they all broke or bent when Mossos showed up. Fixed them today. More will break tomorrow.

Still. 1,863 questions from raw law PDFs in an afternoon. The machine works. It just needs better taste.
